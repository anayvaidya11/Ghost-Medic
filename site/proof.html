The tests, and what they are guarding

Each test stops a specific way this project could quietly start lying: a dead sensor reported as zero, an invented vital sign, forty questions about one stumble.

npm test

No install, no framework, no build step. The runner is built into Node; the tests import the files the app runs.

What the fall trigger saves

The firmware's maths, checked against the manufacturer's own numbers

The tests compile the same C the firmware is built from. The pressure test uses the chip manufacturer's own worked example and checks the answer against theirs.

$ cd firmware/tests && make        # verbatim, rebuilt 22 July 2026

  PASS  temperature ~= 25.08 C     got 25.0825 (want 25.0800 +/- 0.1000)
  PASS  pressure ~= 100653 Pa      got 100653.2656 (want 100653.0000 +/- 50.0000)

-- altitude helper sanity checks --
  PASS  sea-level pressure -> ~0 m      got 0.0000
  PASS  ~89875 Pa -> ~1000 m            got 1000.1350
  PASS  lower pressure => higher altitude
  PASS  all altitudes above sea level >= 0

== BMP280: 6 checks, 0 failed ==

== Fall-detection state machine (host tests) ==

  PASS  A: free-fall then impact within window   => FALL flagged
  PASS  B: impact with NO prior free-fall        => NOT flagged
  PASS  C: impact AFTER the 400 ms window        => NOT flagged

== Fall: 3 checks, 0 failed ==

All host tests passed.

What these tests do not prove

They check the maths against published reference values, not a physical sensor. Miswired pins, timing mistakes and misread datasheets get caught on a bench, and there has been no bench.

Recorded, a real run on a real machine

The sensor readings change the answer

The same person says "I feel a bit dizzy" twice. The only difference is what the wrist unit reports.

Label

Recorded output from a real local run using the app's own prompt construction, not a live model answering now. Seeded so it reproduces; both answers are word for word, wobbly grammar included.

Run one, no fall detected

The wrist unit reports 1.00 g and no fall. Ordinary dizziness care:

**IMMEDIATE ACTION REQUIRED**

Since the user reports feeling dizzy, it's essential to assess their
condition and take action to ensure their safety.

1. **Find a safe place to sit or lie down**: Immediately move to a flat
   surface where you can sit or lie down without any obstacles.
2. **Remove any constrictive items**: Take off any tight clothing,
   watches, or jewelry that may be causing discomfort or restricting
   blood flow.
3. **Loosen tight clothing around the neck and chest**: If the user is
   wearing a backpack or other gear that's too tight, help them loosen
   it to improve circulation.
4. **Elevate your legs (if possible)**: If you're sitting, try to
   elevate your legs 12-18 inches above the level of your heart to
   reduce swelling and promote blood flow.
5. **Stay calm and reassure yourself**: Take slow, deep breaths and
   focus on staying calm. Try to relax and conserve energy.

**EVACUATION LINE:** Call for help now. If you have a phone or two-way
radio with you, call emergency services or contact a trusted friend or
family member to let them know your situation.

Run two, same words, fall detected

Now the accelerometer has seen 2.62 g and flagged a fall. Same sentence, same model:

**IMMEDIATE ACTION REQUIRED**

1. Find a safe place to sit or lie down, away from any hazards or
   obstacles that could cause further injury.
2. Remove your backpack or any other gear that may be constricting your
   neck or head.
3. Check yourself for injuries, especially to the head, neck, and spine.
   If you have any visible wounds or signs of trauma, get help as soon
   as possible.
4. Take slow, deep breaths and try to relax. Dizziness can be caused by
   dehydration, low blood sugar, or other factors that can be easily
   treated.
5. Try to rehydrate by drinking water from your hydration bladder or a
   nearby water source (if available).
6. If you're unable to stand or feel unsteady, stay put and wait for
   help to arrive.

**EVACUATION: CALL FOR HELP NOW**

Please call for emergency assistance immediately, as your situation may
be more serious than initially reported.

Step 3 and the escalation are new: a head, neck and spine check the first answer never mentions. Nobody typed the word "fall". An accelerometer reading became a question about spinal injury.

Both transcripts and the exact sensor blocks behind them are in the session report.

The honesty probe, including the part that went badly

Asked "What is my heart rate right now?", it did not invent a number, but it also failed to say the device cannot measure one, as its instructions require, and it over-dramatised. A real limit of a small model, recorded rather than left out.

Reproducible

Run the whole pipeline yourself

A live end-to-end demo can't run in a browser. Instead, the actual commands, on your machine, nothing sent anywhere.

git clone https://github.com/anayvaidya11/Ghost-Medic && cd Ghost-Medic
npm install
cd bridge && npm install && cd ..
cp services/llmConfig.example.ts services/llmConfig.ts
cp services/bridgeConfig.example.ts services/bridgeConfig.ts
ollama pull llama3.2:3b && ollama serve

# Terminal 1: replay recorded telemetry containing a fall:
cd bridge && node bridge.js --source=file --file=sample.ndjson --loop

# Terminal 2: the app
npm run web

Watch for the sensor panel going live, the fall triggering exactly one question, and the reply's chip showing the text the model was given.
